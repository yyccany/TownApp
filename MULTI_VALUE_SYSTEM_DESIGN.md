# 万物薪俸小镇 - 多元价值观平衡系统设计

## 核心设计原则

### 三条不可逾越的伦理铁律
1. **所有评价只针对自己，绝对不针对别人**
2. **没有绝对的对错，只有价值密度的高低**
3. **用户永远可以自定义自己的价值观**

---

## 一、爱好价值密度系统

### 通用公式
```kotlin
爱好价值密度 = (获得的快乐 + 获得的技能 + 获得的社交 + 获得的收入) / (投入的金钱 + 投入的时间 + 投入的健康)
```

### 评级系统
| 评级 | 价值密度区间 | 说明 |
|------|-------------|------|
| SSS级 | ≥ 100 | 本真的热爱，极高的投入产出比 |
| SS级 | 50-99 | 非常有价值的爱好 |
| S级 | 20-49 | 很好的爱好 |
| A级 | 10-19 | 不错的爱好 |
| B级 | 5-9 | 适度的爱好 |
| C级 | 0.1-4.9 | 可以接受，但可以优化 |
| D级 | < 0.1 | 可能需要重新思考的选择 |

## 二、争议性建筑动态系统

### 建筑状态与价值密度对应
| 价值密度 | 建筑形态 | 小镇影响 |
|---------|---------|---------|
| SSS级 | 大师殿堂 | 大幅正面加成 |
| SS/S级 | 创意工作室 | 中度正面加成 |
| A/B级 | 兴趣小屋 | 轻微正面加成 |
| C级 | 普通房间 | 无影响 |
| D级 | 浪费仓库 | 轻度负面影响 |

### 摄影建筑示例
| 价值密度 | 建筑名称 | 视觉效果 |
|---------|---------|---------|
| SSS级 | 生活摄影馆 | 明亮温馨的房间，墙上挂满照片，NPC开心分享 |
| S级 | 摄影工作室 | 专业但不奢华的工作室，有人在认真创作 |
| C级 | 器材囤积仓库 | 阴暗杂乱，堆满包装未拆的器材 |
| D级 | 器材崇拜教堂 | 奢华的陈列柜，没有人真正使用，只是炫耀 |

## 三、用户自定义价值观系统

### 数据结构
```kotlin
data class UserValueConfig(
    val hobbyWeights: Map<String, HobbyWeight> = emptyMap(),
    val specialValueHobbies: Set<String> = emptySet(),
    val disabledCritiques: Set<String> = emptySet()
)

data class HobbyWeight(
    val joyWeight: Float = 1.0f,
    val skillWeight: Float = 1.0f,
    val socialWeight: Float = 1.0f,
    val incomeWeight: Float = 1.0f
)
```

### 设置页面功能
1. 拖动滑块调整各爱好的价值密度权重
2. 标记"对我有特殊价值"的行为
3. 关闭不认同的批判内容

## 五、社区功能伦理边界

### 三大铁律
1. 禁止晒优越
2. 禁止评判他人
3. 只分享，不劝说

### AI审核关键词
负面关键词：
- "你应该"
- "你怎么还"
- "不如"
- "智商税"
- "浪费"
- "戒了"
- "太"

正面关键词：
- "我觉得"
- "我分享"
- "我体验"
- "对我来说"
- "推荐"

## 六、游戏哲学总结

### 核心信息
> 真正的自由，不是想做什么就做什么。
> 真正的自由，是不想做什么，就可以不做什么。
> 真正的清醒，是知道自己在做什么，知道自己为什么这么做，并且愿意承担所有的后果。

### 游戏态度
> "这是我们的计算结果。
> 但你的人生，你说了算。
> 只要你清楚地知道自己在做什么，并且愿意承担后果，
> 那任何选择都是对的。"

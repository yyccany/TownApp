## 1. 价值密度核心算法

### 1.1 通用计算公式

**文件位置**：`TownSimulationManagerV2.kt`

```kotlin
// 价值密度 = （获得的快乐 + 获得的技能 + 获得的社交 + 获得的收入） / 
//           （投入的金钱 + 投入的时间 + 投入的健康）
valueDensity = (joy + skill + social + income) / (money + time + health)
```

### 1.2 7级评判标准

| 等级 | 价值密度区间 | 建筑表现 |
|------|------------|---------|
| SSS级 | ≥100 | 辉煌、完美 |
| SS级 | ≥50 | 优秀、光明 |
| S级 | ≥20 | 良好、正面 |
| B级 | ≥5 | 普通、中性 |
| C级 | ≥0.1 | 较差、负面 |
| D级 | ≥0.001 | 很差、灰暗 |
| F级 | <0.001 | 废弃、灾难性 |

### 1.3 共识性例外规则

**永远正面的行为**（不受价值密度影响）：
- 运动/健身
- 阅读/学习
- 规律睡眠

**永远负面的行为**（不受价值密度影响）：
- 吸烟
- 酗酒
- 吸毒

## 2. 争议性建筑动态逻辑

### 2.1 建筑变化维度

**文件位置**：`HobbyConfigs.kt`

| 维度 | 变化规则 |
|------|---------|
| **结构** | 模块化部件替换，从下到上 |
| **材质** | 参数化调整颜色、金属度、粗糙度 |
| **环境** | 周边植被、垃圾、灯光联动变化 |
| **NPC** | NPC行为、表情、对话同步变化 |

## 3. 多结局系统规则

### 3.1 结局判定优先级

**文件位置**：`EndingSystem.kt`

结局按优先级从高到低检查：
1. 觉醒类（觉醒值≥10000）
2. 艺术创作类（最高价值密度爱好≥100）
3. 游戏开发类（同上）
4. 旅行探索类（同上）
5. 运动健身类（同上）
6. 极简生活类（极简分≥1000）
7. 家庭守护类（家庭分≥1000）
8. 知识学习类（同上）
9. 社交蝴蝶类（社交分≥1000）
10. 商业创业类（爱好收入≥100）
11. 自然环保类（自然分≥1000）
12. 精神修行类（精神分≥1000）
13. 美食烹饪类（同上）
14. 平凡稳定类（觉醒值1000-9999）
15. 生活探索类（高价值多样性≥5）
16. 迷失反思类（觉醒值<1000）

### 3.2 结局平等原则

**重要！**：所有结局视觉效果同等精美，没有"好结局"和"坏结局"的颜色区分。

## 4. NPC状态系统

### 4.1 三层变量体系

| 变量层 | 说明 | 变化频率 |
|-------|------|---------|
| **基础属性** | 体力、智力、魅力、自律、创造、财富上限 | 固定不变 |
| **小镇关联变量** | 与建筑状态、环境数值联动 | 每小时更新 |
| **用户行为关联变量** | 通过价值密度→小镇状态→NPC状态链路间接影响 | 实时更新 |

### 4.2 行为树架构

采用分层行为树设计，避免硬编码：
```
Root
├─ Check Need (生存需求)
├─ Check Emotion (情绪状态)
├─ Check Environment (环境影响)
└─ Execute Action (执行行为)
```

## 💡 迁移提示

### Kotlin → C# 快速对照表

| Kotlin | C# |

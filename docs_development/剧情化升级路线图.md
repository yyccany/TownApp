# 小镇剧情化升级路线图（对标《去月球》《七年后等着你》）

> 本文档梳理小镇项目**对标两款像素剧情神作**的差距与落地节奏。
> 核心原则：**依托现有 NPC 分层架构增量开发，不推翻现有代码**。

---

## 一、核心叙事底层：时间是唯一主线

### 现有基础
✅ 天数/四季/年龄数值系统
✅ NPC 好感分层
✅ 认知觉醒系统
✅ 时间银行统计逻辑

### 1.1 时间不可逆 + 人生分支锁死机制

**核心理念**：时间是小镇唯一货币，不可回滚。

**落地内容**：
- 错过固定年龄段专属事件永久关闭
  - 18岁求职：仅在 18-22 岁窗口期可触发
  - 30岁成家：仅在 28-32 岁窗口期可触发
  - 50岁健康拐点：50岁后健康不可逆下滑
- NPC 人生节点与玩家时间强绑定
  - 玩家到 40 岁，NPC 同步变老、离职、患病、退休
  - 无法回头重刷年轻时剧情
- 时间消耗量化代价
  - 每度过一年，永久损耗健康/认知，不能靠操作回滚

**工程落地**：
- `AgeWindow` 内存模型管理窗口期判断
- `NpcLifecycle` 自动迭代逻辑（每年 NPC 独立推进）
- `TimeConsumptionCalculator` 统一计算年度消耗

### 1.2 长线跨年份伏笔回收系统

**核心理念**：早年不经意的对话，微小选择，几十年后产生巨大影响。

**落地内容**：
- 新增 `TimePlotMark` 内存模型
  - 存储玩家任意时间点的选择ID、NPC对话标记
  - 玩家晚年叙事自动读取早年伏笔，生成呼应式独白

**工程落地**：
- `TimePlotMarkRepository` 统一管理伏笔库
- `MemoryRecallEngine` 根据伏笔匹配晚年独白
- 伏笔存储：仅存 ID 数字，文案走 `TextAssetLoader` 静态文本

### 1.3 四季 = 人生阶段深度差异化叙事池

**核心理念**：四季对应人生阶段，对话、剧情、价值判断完全不同。

**落地内容**：

| 季节 | 人生阶段 | 叙事核心 | 认知手术刀频率 |
|------|---------|---------|--------------|
| 春 | 青年（18-30）| 迷茫试错：求职、理想、爱情试探 | 低 |
| 夏 | 壮年（30-45）| 高压生存：内卷、房贷、医疗、职场 | 中 |
| 秋 | 中年（45-60）| 得失反思：人生下半场、健康拐点 | 高 |
| 冬 | 晚年（60+）| 宿命复盘：时间银行结算、人生结局 | 最高 |

**工程落地**：
- assets/text/narration/{spring|summer|autumn|winter}/*.json
- `SeasonalNarrativeLoader` 按年龄 + 季节加载对应文案
- 全部走 `TextAssetLoader`，不污染数据层

### 1.4 记忆碎片化叙事

**核心理念**：走到特定地点、见到特定 NPC，自动弹出多年前片段式短剧情。

**工程落地**：
- `MemoryFragmentDialog` Compose 组件
  - 按需弹出、自动销毁
  - 不常驻界面，符合极简 UI 原则
  - 接收 ViewModel 给到的现成文本，无 ID 匹配逻辑

## 二、NPC & 人际系统：完整独立人生

### 现有基础
✅ NpcBase / NpcStatus 内存骨架
✅ 职业区分对话
✅ 好感分层 ID 匹配
✅ NpcRepository 多条件查询收口

### 2.1 NPC 独立时间线

**核心理念**：每个 NPC 拥有完整一生，玩家只是闯入者。

**落地内容**：
- NPC 年度自动迭代逻辑
  - 无需玩家互动，每过游戏一年，NPC 自动触发：跳槽、生病、结婚、衰老、离世
  - 玩家错过相遇窗口，NPC 人生直接走向另一条分支
- NPC 四维数值独立运算
  - 和玩家时间同步流转
  - 不依附玩家存在

**工程落地**：
- `NpcLifecycleManager`：每年度自动调用，遍历所有 NPC 推进状态
- `NpcLifeEvent`：NPC 独立事件类型（跳槽/生病/结婚/衰老/离世）
- 存储于 `NpcStatus` 现有字段，扩展生命周期数字标记

### 2.2 深度羁绊支线剧情链

**核心理念**：陌生→相识→共事→交心→别离，每阶段专属剧情。

**落地内容**：
- 五阶段羁绊系统
  1. 陌生：浅层对话，无支线
  2. 相识：触发支线序幕
  3. 共事：支线展开
  4. 交心：深层对话解锁
  5. 别离：结局分支
- 每阶段配套：专属剧情、道具、记忆碎片

**工程落地**：
- `NpcStatus.intimacyStage: Int` 标识羁绊阶段
- assets/text/npc/bond/{npcId}/stage_{1-5}.json
- `NpcBondRepository` 统一管理支线进度

### 2.3 角色矛盾与观念冲突

**核心理念**：不同年龄、职业 NPC 自带成长环境塑造的认知，与玩家认知等级形成对话对抗。

**落地内容**：
- 玩家认知数值越高，解锁理性辩证对话
- 低认知只会浅层争吵
- 复刻现实中"两代人互相不理解"的内核

**工程落地**：
- `NpcDialogTrigger` 已支持 favor 分级，扩展 `cognitionLevel: Int` 维度
- assets/npc/dialog/{season}/ 目录下按 cognition_level 分目录
- 不增加数据库负担，仅扩展文本匹配规则

## 三、视听沉浸层：像素氛围营造

### 现有基础
✅ 简易 MusicManager 音频骨架
✅ 像素 UI 配色规范
✅ 静态素材目录规划

### 3.1 分时段动态 BGM + 场景音效联动

**核心理念**：音乐是情绪的支柱，动态切换才能营造氛围。

**落地内容**：
- 时段切换 BGM
  - 清晨（6-10）：轻柔钢琴
  - 白天（10-18）：轻快日常
  - 傍晚（18-20）：舒缓吉他
  - 深夜（20-6）：低沉独处纯音
- 剧情触发瞬时切专属情绪曲
  - 裁员：紧张压抑短 BGM
  - 重病：低沉吟唱
  - 重逢：温暖回忆
  - 临终复盘：钢琴 + 大提琴
- 环境持续音效（分层混音）
  - 街道车流、办公室键盘、居家白噪音

**工程落地**：
- `MusicManager` 扩展为分时段调度
- `EmotionMusicTrigger`：根据 ViewModel 事件触发专属 BGM
- 音频资源位于 `assets/music/{period|emotion|ambient}/`

### 3.2 Compose 像素镜头运镜系统

**核心理念**：画面氛围感靠镜头语言。

**落地内容**：
- 慢推进镜头
  - 进入晚年、关键回忆时画面缓慢放大
- 淡入淡出转场
  - 昼夜交替、四季更替、回忆切入切出
- 视觉特效
  - 轻微画面抖动：角色生病时
  - 低饱和度滤镜：精神崩溃时
  - 黑白渐变：回忆场景

**工程落地**：
- 通用 `Modifier` 扩展封装镜头效果
  - `Modifier.slowZoomIn()` / `Modifier.fadeTransition()` / `Modifier.shakeEffect()`
- 地图页面直接复用，不打乱现有分层
- 状态由 ViewModel 推送，UI 自动应用效果

### 3.3 像素动态细节动画

**核心理念**：静态画面转动态才有生命力。

**落地内容**：
- 角色行走、眨眼帧动画
- 四季场景环境变化
  - 春天花开、夏天蝉鸣、秋天落叶、冬天落雪
- 时间流逝光影渐变

**工程落地**：
- 静态素材拆分帧动画
- Canvas 渲染，不增加数据库负担
- `assets/sprites/{npc|environment}/` 帧序列图

## 四、剧情框架与结局体系：完整闭环

### 现有基础
✅ 16 种结局规划
✅ 认知觉醒复盘面板
✅ 时间银行统计

### 4.1 时间加权结局计算

**核心理念**：结局由一生全部时间行为加权计算，不是单一选择判定。

**落地内容**：
- 全局时间行为统计模型
  - 数十年工作时长、陪伴 NPC 时长、休息消耗
  - 健康损耗、认知解锁程度
  - 加权算出专属人生结局
- 真正落地"时间是唯一货币"

**工程落地**：
- `LifeEndingCalculator` 全局计算器
- 汇总各类时间投入数据，按权重算出结局 ID
- 走 `TextAssetLoader` 加载结局文案

### 4.2 多周目时间轮回机制

**核心理念**：通关后解锁二周目记忆继承，拔高时间主题深度。

**落地内容**：
- 通关结局后解锁二周目
  - 保留：认知等级、时间复盘记录
  - 清空：所有财富、NPC 好感
  - 玩家带着前世认知重新度过一生
- 形成对比叙事
  - 同样的事件，不同的选择
  - 产生独属于玩家的"如果当初..."感悟

**工程落地**：
- 周目数据单独存入内存仓库
- `RoundDataManager` 管理周目隔离
- 单局存档和周目存档完全分离

### 4.3 无标准答案的价值思辨叙事

**核心理念**：所有结局不分好坏，仅客观展示不同时间分配的代价。

**落地内容**：
- 时间银行、认知手术刀完整拆解每条人生道路的得失
- 不灌输单一价值观
- 区别于商业剧情游戏的独特核心竞争力
- 符合小镇"自由平等、实事求是"内核

**工程落地**：
- 结局文案采用"得 + 失"对照格式
- `LifeArchiveScreen` 可视化展示每条选择的时间代价
- 复盘页面对接认知手术刀组件

### 5.1 完善 TextAssetLoader 全量叙事文本缓存

- 统一管理数万条时间、NPC、回忆文案
- 彻底隔离数据层与长文本

## 七、对现有架构的影响评估

| 模块 | 现有架构 | 新增内容 | 是否需要重构 |
|------|---------|---------|------------|
| NPC 数据层 | NpcBase/NpcStatus/NpcDao/NpcRepository | 扩展 lifecycle/bond 字段 | ❌ 增量扩展 |
| 文本加载 | TextAssetLoader | 增加季节/年龄段/认知维度 | ❌ 仅加文件 |
| UI 组件 | NpcTownMapScreen | 加 MemoryFragmentDialog | ❌ 新增组件 |
| ViewModel | NpcViewModel | 加事件回调 | ❌ 加方法 |
| 音频 | MusicManager | 分时段调度 | ❌ 扩展方法 |
| 视觉 | Compose 通用 Modifier | 加运镜效果 | ❌ 新增 Modifier |

**结论**：所有升级均**增量开发**，不推翻现有代码，符合小镇项目"永远增量更新，禁止推倒重构"的铁律。

## 八、与小镇三大边界的契合度

### 边界一：永不加入焦虑驱动设计
✅ 时间不可逆 = 引导玩家珍惜当下，不催促完成任务  
✅ 无排行榜、无签到 = 不绑架注意力  

### 边界二：永不输出评判性结论
✅ 结局只展示得失 = 不定义"成功人生"  
✅ 复盘拆解客观 = 不灌输单一价值观  

### 边界三：永不强迫用户行动
✅ 错过窗口 = 自然后果，不是惩罚  
✅ NPC 独立 = 玩家不被强制交互  

### 方向校验
**"它是在增加'我要足够好才配'的焦虑，还是在强化'你这样就很好'的底气？"**  
→ 时间叙事让玩家看到：每一种活法都有得有失，**你的人生就是完整的**。

# 消费系统深化开发文档（v2.0）

## 一、概述

### 1.1 核心目标
把"钱花在刀刃上"从抽象概念变成可感知的游戏体验。

### 1.2 设计原则
1. **无对错评判** - 只用客观数值展示不同取舍的长期结果
2. **外物只是工具** - 美食/服饰/装修/车辆只是工具，人的身心才是核心资源
3. **两种路线都能走完人生** - 只是长期状态不同
4. **数据默认折叠** - 体现无评判的治愈内核，不制造焦虑

### 1.3 遵循小镇宪章
- ✅ 永不加入焦虑驱动设计
- ✅ 永不输出评判性结论
- ✅ 永不强迫用户行动

## 三、数据模型详解

### 3.1 UserConsumptionStateEntity（用户消费状态）

#### 四大消费维度评分（0~100）
| 维度 | 字段 | 人本路线（高分） | 虚荣路线（低分） |
|-----|------|----------------|----------------|
| 吃 | foodScore | 日常优质食材 | 宴席网红溢价 |
| 穿 | clothingScore | 贴身舒适 | 大牌logo |
| 住 | housingScore | 环保健康家电 | 网红软装颜值 |
| 行 | transportScore | 安全护具 | 豪车排面贷款 |

#### 累积统计数据
| 字段 | 类型 | 说明 |
|-----|------|------|
| peopleOrientedCount | Int | 累计人本消费次数 |
| vanityDrivenCount | Int | 累计虚荣消费次数 |
| totalConsumptionAmount | Double | 累计消费金额 |
| savedTimeHours | Double | 累计节省的医疗/维修/还贷时间 |

#### 计算属性
| 属性 | 类型 | 说明 |
|-----|------|------|
| overallScore | Int | 综合人本取向评分（四者平均） |
| isPeopleOriented | Boolean | 是否整体偏向人本路线（≥60） |
| isVanityDriven | Boolean | 是否整体偏向虚荣透支路线（≤40） |

### 3.2 ConsumptionChoiceEventEntity（消费抉择事件）

| 字段 | 类型 | 说明 |
|-----|------|------|
| eventId | String | 事件ID（对应文案资源） |
| eventType | String | 事件类型（food/clothing/housing/transport） |
| triggerAge | Int | 触发时的玩家年龄 |
| playerChoice | Int | 玩家的选择：0=未选择，1=人本路线，2=虚荣路线 |
| isTriggered | Boolean | 是否已触发 |

## 四、核心业务逻辑

### 4.1 评分更新规则

#### 单次消费的评分影响
- 人本消费：+2~+5分（取决于消费类型和重要性）
- 虚荣消费：-2~-5分
- 评分范围：0~100，不会溢出

#### 不同维度的权重
| 维度 | 单次抉择影响 | 说明 |
|-----|------------|------|
| 吃 | ±3分 | 高频消费，影响较小 |
| 穿 | ±3分 | 中频消费，影响适中 |
| 住 | ±8分 | 低频大额消费，影响较大 |
| 行 | ±6分 | 低频大额消费，影响较大 |

### 4.2 长期效果模型（LongTermEffect）

基于当前消费倾向评分，计算长期累积的身心状态修正。

| 效果字段 | 说明 | 人本路线（高分） | 虚荣路线（低分） |
|---------|------|----------------|----------------|
| healthDeltaPerDay | 每日健康增减 | 正（健康提升） | 负（健康下降） |
| moodDeltaPerDay | 每日情绪增减 | 正（情绪稳定） | 负（情绪波动） |
| fatigueMultiplier | 精力消耗倍率 | <1（更轻松） | >1（更累） |
| sleepQualityDelta | 睡眠质量修正 | 正（睡得好） | 负（睡得差） |
| illnessProbabilityMultiplier | 生病概率倍率 | <1（少生病） | >1（易生病） |
| freeTimeBonusHoursPerWeek | 每周额外自由时间 | 正（时间多） | 负（时间少） |

### 4.3 消费抉择事件系统

#### 触发时机
- 特定年龄节点（25岁、30岁、35岁、40岁等）
- 特定消费场景（买房、买车、装修等大额消费）
- 随机触发（低概率，增加游戏丰富度）

#### 抉择流程
1. 事件触发 → 弹出抉择对话框
2. 玩家选择（人本路线 / 虚荣路线）
3. 更新对应维度的评分
4. 记录消费类型统计
5. 长期效果逐步累积

#### 设计特点
1. **数据默认折叠** - 只显示综合评分，详细数据需要点击展开
2. **温柔的色调** - 绿色（人本）、黄色（中立）、橙色（虚荣），不使用红色
3. **无评判语言** - 只展示客观数值，不用"好/坏"、"对/错"等评判性词汇
4. **卡片式布局** - 每个模块独立卡片，视觉清晰

## 九、注意事项

### 9.1 价值观红线
1. 绝对不能出现"正确消费观"、"理性消费"等评判性表述
2. 两种路线都是完整的人生，没有优劣之分
3. 人本路线也有缺点（比如社交场合可能被看不起）
4. 虚荣路线也有优点（比如短期情绪满足、社交资本）

### 9.2 反焦虑设计
1. 数据默认折叠，不主动展示
2. 没有排行榜、没有成就系统
3. 没有"消费健康度"之类的评判性指标
4. 温柔的色调和文案，不制造紧迫感

### 9.3 工程规范
1. 严格遵循MVVM架构
2. 数据驱动与内容分离，文案走资源层
3. 所有状态不可变，通过copy()生成新实例
4. StateFlow驱动UI，单向数据流

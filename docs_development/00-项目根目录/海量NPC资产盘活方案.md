# 海量 NPC 资产盘活方案：众生相差异化视觉+叙事

> 本文档是「产品终极定位」战略的**执行落地版本**。
> 核心目标：**把存量文本/人物数据转化为「众生相差异化视觉+叙事」**。
> 原则：**不新增文案，只改造渲染逻辑；零底层重构，全模块复用**。

---

## 一、核心思路：盘活存量，不造新轮子

### 现状资产盘点（已落地）

| 资产 | 位置 | 状态 |
|------|------|------|
| 数十位独立 NPC（职业、年龄、性格、命运各异） | NpcBase | ✅ 已落地 |
| 分职业衣食住行对话台词 | assets/text/npc/dialog/*.json | ✅ 已落地 |
| 完整生命周期（求职/中年/退休/衰老/离世） | NpcLifecycleManager | ✅ 已落地 |
| 分年龄段回忆碎片（5/10/20/30 年节点） | assets/text/npc/memory/*.json | ✅ 已落地 |
| 差异化价值观对话 | 分级对话 ID 匹配 | ✅ 已落地 |
| 季节/职业/好感组合对话 | Repository 多条件查询 | ✅ 已落地 |

### 核心痛点

> **海量人设素材只是单纯堆在静态 JSON 里，界面没有把这份独特性释放出来。**
> **所以看不出和农场的差别。**

### 解决思路

**视觉层 + 叙事层 + 交互层**三路并进，**全部复用现有数据**，仅改渲染逻辑。

## 二、视觉层：NPC 专属氛围色调系统

### 1. 数据层仅新增数字标识

**在 NpcBase 增加 3 个数字字段**（无需新增文本，零数据库压力）：

```kotlin
val tonePaletteId: Int,        // 专属色调组 ID
val vignetteStrength: Float,    // 专属暗角强度
val saturationBias: Float       // 饱和度偏移
```

**新建静态资源配置**：

```
assets/text/npc/tone_palette.json
```

仅存储色值参数，**不写任何剧情文字**。

### 2. 氛围色与职业/性格对应表

| tonePaletteId | 职业/性格 | 主色调 | 暗角 | 饱和度 |
|---------------|----------|-------|------|--------|
| 1 | 教师 | 温柔米白柔光 | 0.15 | 0.85 |
| 2 | 工人 | 质朴灰黄 | 0.20 | 0.90 |
| 3 | 医生 | 冷静浅蓝 | 0.10 | 0.95 |
| 4 | 创业者 | 高对比亮金 | 0.00 | 1.20 |
| 5 | 农民 | 大地暖棕 | 0.25 | 0.80 |
| 6 | 晚年老人 | 低饱和暖棕褪色 | 0.40 | 0.60 |
| 7 | 失意者 | 暗角低亮度 | 0.60 | 0.50 |
| 8 | 奋斗者 | 通透高光 | 0.05 | 1.15 |
| 9 | 治愈系邻居 | 淡绿柔光 | 0.10 | 0.95 |
| 10 | 病休 NPC | 灰白低对比 | 0.50 | 0.40 |

### 3. 画面层级顺序调整

**新增一层「NPC 专属人物氛围层」**：

```
1. 像素画布（最底层）
2. 全局季节昼夜遮罩 AtmosphereOverlay
3. NPC 专属人物氛围滤镜 ⬅️ 新增
4. 悬浮 UI（MiniStatusPanel / FloatingMenu）
5. 弹窗（对话 / 回忆）
```

### 4. 触发逻辑

```kotlin
// 玩家点击 NPC 触发对话/回忆
val selectedNpc = viewModel.selectedNpc.collectAsState().value
NpcMoodOverlay(
    palette = if (selectedNpc != null) 
        repo.getMoodPalette(selectedNpc.npcId) 
    else null  // null = 不激活，回退到全局基础色
)
```

### 5. 落地效果

| 场景 | 画面氛围 |
|------|---------|
| 和退休老教师对话 | 画面铺满柔和米白柔光，饱和度降低，暗角轻微 |
| 和加班创业者对话 | 画面高光拉高，高对比金色调，无暗角 |
| 和常年疲惫工人对话 | 灰黄低亮度，四周加深暗角 |
| 走近病休 NPC | 灰白低对比，四周明显暗角 |

> **仅凭画面光影，不用看文字，玩家就能直观感知这个人的生活状态。**

## 三、叙事层：NPC 自主日常行为系统

### 1. 现有问题：农场交互逻辑

- 静止小人
- 点击弹出一段固定文字
- **点击才有反馈**

### 2. 差异化逻辑：NPC 自带独立日常行为循环

> **衣食住行台词随自身行为实时刷新，不等待玩家点击。**

### 3. 落地步骤

#### 步骤 1：新增每日行为调度工具类

**新建 `NpcDailyBehaviorScheduler.kt`**

- 每个 NPC 每日自动分配行为 ID
- 行为类型：上班、买菜、散步、居家做饭、看病、独处发呆
- 行为 ID 绑定 assets 内对应的衣食住行短句库

#### 步骤 2：Canvas 实时渲染 NPC 动态帧动画

| 行为 | 帧动画 |
|------|--------|
| 上班 | 行走动画（早 8 点） |
| 买菜 | 缓慢走动（午 12 点） |
| 散步 | 缓慢踱步（傍晚 18 点） |
| 居家做饭 | 停在房屋门口（晚 19 点） |
| 看病 | 步履迟缓（随机） |
| 独处发呆 | 静止+随机眨眼（深夜） |

#### 步骤 3：画面边角漂浮极淡半透明小字独白

**完全自主刷新，玩家路过就能瞥见 TA 当下的生活感慨**：

- 工人 NPC 买菜时浮动小字：「今天菜价又涨了，每月房租压得喘不过气」
- 教师傍晚散步：「放学看着孩子们，心里总能轻松一点」
- 创业者加班时：「这个项目成了就能翻盘，再拼一把」

**视觉规范**：
- 半透明（alpha 0.4-0.6）
- 极小字号（10sp）
- 边缘漂浮（Canvas 边角）
- 不遮挡主画面

#### 步骤 4：点击 NPC 才展开完整分层对话

**对话三层结构**（复用现有台词）：

| 层级 | 内容 | 加载方式 |
|------|------|---------|
| 浅层 | 三餐、通勤、物价等生活化闲聊 | dialog/{season}.json |
| 中层 | 工作压力、人际、理想与现实的冲突 | 对话 ID + 好感分级 |
| 深层 | 跨年份回忆、人生遗憾与收获 | memory/*.json |

**三段文本全部是现有存量资源，仅调整加载展示逻辑，无需新增文案。**

## 四、档案馆层：众生时间档案馆

### 1. 核心定位

> **《去月球》只能看一对人的一生。**
> **你拥有全镇所有人数十年的人生记录。**
> **这是独一无二的核心竞争力。**

### 2. 落地步骤

#### 步骤 1：扩展右下角折叠菜单

**在 `FloatingExpandMenu` 中新增「小镇人生档案馆」入口**：

```kotlin
MenuItem(
    label = "人生档案馆",
    icon = Icons.Filled.AccountBox,
    onClick = { viewModel.openArchive() }
)
```

#### 步骤 2：档案馆内 NPC 分类系统

**按维度分类浏览**：

| 分类维度 | 选项 |
|---------|------|
| 片区 | 商业街/住宅区/学校/医院/农田 |
| 职业 | 教师/工人/医生/创业者/农民/店主 |
| 年龄段 | 青年(20-30) / 壮年(30-45) / 中年(45-60) / 晚年(60+) |
| 好感度 | 陌生人/熟人/朋友/挚友 |

#### 步骤 3：点开任意人物，串联完整时间线

**自动串联该 NPC 从青年到晚年的完整时间线**：

1. 早年求职衣食拮据对话（dialog/job_1_season_1_low.json）
2. 中年家庭与工作矛盾（dialog/job_1_season_3_mid.json）
3. 晚年回望一生的独白（memory/memory_xxx.json）
4. 工作状态变化（在职 → 退休 → 病休 → 离世）

#### 步骤 4：时间线穿插自动触发的记忆碎片弹窗

**复用 `MemoryFragmentDialog` 组件**：
- 翻阅某人档案时
- 跨年份节点（5/10/20/30 年）自动触发
- 弹出对应 markId 的回忆

> **玩家可以抛开自身主线，单独翻阅任意陌生人完整一生。**
> **实现"一人体验千百种人生"的产品定位。**

### 3. 档案馆主屏幕结构

```
人生档案馆
├── 顶部：分类筛选（片区/职业/年龄段/好感）
├── 主体：NPC 列表（缩略头像 + 姓名 + 职业 + 年龄 + 好感分级）
└── 详情：点击进入个人时间线（对话+回忆+状态变化）
```

## 五、交互层：弱化农场感，强化观察者身份

### 1. 移除所有功能性操作按钮

**玩家核心行为只有两种**：
- **漫步观察**（在地图自由移动）
- **驻足交谈**（点击 NPC 触发对话）

### 2. 长时间停留自动弹出微量日常独白

- 玩家在某 NPC 身旁停留 > 10 秒
- 自动弹出极淡半透明小字独白
- **不用点击，鼓励沉浸式观察小镇众生百态**

### 3. 对话分层贴合衣食住行人设

| 层级 | 内容 | 触发方式 |
|------|------|---------|
| 浅层 | 三餐、通勤、物价等生活化闲聊 | 首次对话 |
| 中层 | 工作压力、人际、理想与现实的冲突 | 好感 ≥ 30 |
| 深层 | 跨年份回忆、人生遗憾与收获 | 好感 ≥ 70 + 触发回忆 mark |

### 4. 无任务、无目标、无进度条

✅ **不催促玩家交互**
✅ **贴合项目"不制造焦虑"的底层边界**
✅ **和经营农场的任务驱动彻底切割**

## 七、最终差异化呈现

| 作品 | 角色数量 | 角色深度 | 视觉差异化 | 玩家角色 |
|------|---------|---------|----------|---------|

| QQ 农场 | 数十工具人 | 无 | 全部同质 | 农场主 |
| **小镇（本项目）** | **数十完整人** | **深（数十人）** | **每人专属光影** | **观察者** |

### 改造完成后的小镇体验

> **数十位拥有完整衣食住行、独立人生轨迹的普通人自主生活，每个人拥有专属光影气质，玩家可自由观察全镇百态，翻阅任意人的一生。**

### 核心价值

✅ **海量 NPC 与人设文本不再是后台闲置素材**
✅ **直观支撑"单人亲历人间洪流"核心定位**
✅ **形成无法复刻的产品壁垒**

## 八、与小镇三大边界的契合度

### 边界一：永不加入焦虑驱动设计

✅ 档案馆无时间限制，想看谁看谁
✅ 自主动画不催促玩家
✅ 无任务、无目标、无进度条

### 边界二：永不输出评判性结论

✅ 失意/奋斗/治愈只是不同氛围色
✅ 不评论哪种人生更好
✅ 客观呈现各种生活状态

### 边界三：永不强迫用户行动

✅ 漫步观察 = 零操作
✅ 自主动画 = 不点击也有内容
✅ 对话分层 = 可只看浅层闲聊

### 方向校验

> **"它是在增加'我要足够好才配'的焦虑，还是在强化'你这样就很好'的底气？"**

✅ **每种活法都有自己的人生质感**
✅ **每种光影都值得被看见**
✅ **你不是主角，你是千万普通人中的一个观察者**
✅ **你的人生就是完整的，无需与他人比较**

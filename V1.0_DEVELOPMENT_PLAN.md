## 🏗️ 数据库架构设计

### 核心表结构

#### 1. 用户基本信息表 (user)
```sql
CREATE TABLE user (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    hourly_wage REAL NOT NULL DEFAULT 28.4,
    awakening_value INTEGER NOT NULL DEFAULT 0,
    health_value INTEGER NOT NULL DEFAULT 100,
    happiness_value INTEGER NOT NULL DEFAULT 50,
    debt_value INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. 行为记录表 (actions)
```sql
CREATE TABLE actions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type TEXT NOT NULL, -- "food", "clothing", "housing", "mental", "learning"
    sub_type TEXT NOT NULL, -- 具体类型
    amount REAL NOT NULL, -- 金额/时长/数量
    metadata TEXT, -- JSON格式的附加数据
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### 3. 建筑表 (buildings)
```sql
CREATE TABLE buildings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    building_id TEXT NOT NULL, -- 对应TownRepository中的ID
    district TEXT NOT NULL, -- "digestive", "skin", "habitat", "spirit"
    level INTEGER NOT NULL DEFAULT 1,
    position_x REAL,
    position_y REAL,
    is_active BOOLEAN NOT NULL DEFAULT 1,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    last_updated TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### 4. NPC表 (npcs)
```sql
CREATE TABLE npcs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    npc_id TEXT NOT NULL,
    name TEXT NOT NULL,
    role TEXT NOT NULL,
    is_negative BOOLEAN NOT NULL,
    avatar TEXT,
    current_state TEXT, -- "working", "resting", "walking"
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### 5. 事件历史表 (event_history)
```sql
CREATE TABLE event_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    event_id TEXT NOT NULL,
    event_name TEXT NOT NULL,
    is_negative BOOLEAN NOT NULL,
    effects TEXT NOT NULL, -- JSON格式的效果
    triggered_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    duration_days INTEGER NOT NULL DEFAULT 1
);
```

#### 6. 解锁成就表 (unlocks)
```sql
CREATE TABLE unlocks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    type TEXT NOT NULL, -- "idiom", "library_section", "achievement"
    content_id TEXT NOT NULL,
    unlocked BOOLEAN NOT NULL DEFAULT 0,
    unlocked_at TIMESTAMP,
    UNIQUE(type, content_id)
);
```

#### 7. 每日任务表 (daily_tasks)
```sql
CREATE TABLE daily_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL, -- "YYYY-MM-DD"
    task_type TEXT NOT NULL,
    description TEXT NOT NULL,
    completed BOOLEAN NOT NULL DEFAULT 0,
    completed_at TIMESTAMP,
    UNIQUE(date, task_type)
);
```

## 🎮 小镇动态模拟规则

### 建筑生成规则

#### 负面建筑触发条件
```kotlin
// 糖衣炮弹工厂
when {
    weeklySugaryDrinks >= 7 -> Level 1
    weeklySugaryDrinks >= 14 -> Level 2
    weeklySugaryDrinks >= 21 -> Level 3
}

// 短视频收割塔
when {
    dailyShortVideoHours >= 2 -> Level 1
    dailyShortVideoHours >= 4 -> Level 2
    dailyShortVideoHours >= 6 -> Level 3
}

// 人情银行
when {
    annualGiftMoney >= 3000 -> Level 1
    annualGiftMoney >= 6000 -> Level 2
    annualGiftMoney >= 10000 -> Level 3
}
```

#### 正面建筑触发条件
```kotlin
// 徒步俱乐部
when {
    weeklyExerciseHours >= 3 -> Level 1
    weeklyExerciseHours >= 7 -> Level 2
    weeklyExerciseHours >= 14 -> Level 3
}

// 断舍离回收站
when {
    monthlyDiscardedItems >= 10 -> Level 1
    monthlyDiscardedItems >= 30 -> Level 2
    monthlyDiscardedItems >= 50 -> Level 3
}
```

#### 建筑消失规则
```kotlin
// 负面建筑连续30天无触发行为 → 逐渐消失
// Stage 1: 停止冒烟/发光（第1-10天）
// Stage 2: 建筑开始褪色（第11-20天）
// Stage 3: 建筑倒塌消失（第21-30天）
```

### NPC系统
```kotlin
// NPC生成规则
- 每个新建筑生成2-3个对应NPC
- 负面建筑生成负面NPC
- 正面建筑生成正面NPC

// NPC对话库
// 每个NPC有3-5个随机对话

// NPC迁移规则
- 负面建筑消失 → 负面NPC离开小镇
- 正面建筑升级 → 新正面NPC迁入
```

### 事件系统
```kotlin
// 定时事件
- 每天8:00生成每日小事任务
- 每周一更新本周目标

// 阈值事件
- 连续7天刷短视频 >4小时 → 注意力危机
- 每月打赏 >1000元 → 打赏狂欢夜
- 连续30天不喝含糖饮料 → 健康日

// 正面事件
- 觉醒值突破100 → 觉醒日
- 连续完成7天任务 → 坚持就是胜利
```

│   ├── 文言文：最大的时间浪费（解锁：记录语言学习）
